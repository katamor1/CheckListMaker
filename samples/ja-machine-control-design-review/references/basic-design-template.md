# 基本設計書テンプレート

文書情報には、次の項目を記載する。

- 文書管理番号: `DMS-####` 形式（必須）
- 文書名: 対象機能を識別できる名称（必須）
- 版: レビュー対象の版（必須）
- 機密区分: 許可された区分（必須）

## 1. 目的

設計対象の目的と、レビューで確認できる達成事項を具体的に記載する。

## 2. 適用範囲

`対象` と `除外` の両方を必ず明示する。

- 対象: 設計およびレビューに含める機能、装置、処理
- 除外: 設計およびレビューに含めない機能、装置、処理

## 3. 構成

構成要素、責務、要素間の関係、および外部インターフェースを記載する。

## 4. 機能設計

入力、処理、出力、主要パラメータ、制約、および正常時の動作を記載する。

## 5. 異常処理

検出条件、通知、停止または縮退の動作、復旧条件、および記録方法を記載する。

## 6. スケジュール

設計、レビュー、承認、改訂の予定と基準日を記載する。

## 7. 承認

次の項目を明示する。

- 改訂日: YYYY-MM-DD 形式（必須）
- 承認者: 氏名または承認責任者の識別名（必須）
- 承認日: YYYY-MM-DD 形式
- 承認手順: レビュー完了から承認までの手順
