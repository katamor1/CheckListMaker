## 確認項目

- [ ] ホーム画面の見出しとバージョン情報に、利用者向けの英語装飾見出しが残っていない。
- [ ] 新規プロジェクトのボタンが「既存文書の検証を開始」「文書生成と検証を開始」と表示される。
- [ ] 既存文書モードで「主対象文書が選択されていません。」と明示される。
- [ ] 文書生成モードの入力項目名が、承認済みの用語と一致する。
- [ ] 参考資料、チェックリスト、チェック項目、条件、評価対象範囲で用語が統一されている。
- [ ] 事前検査のエラーコードが、詳細を開く前は表示されない。
- [ ] 事前検査と利用者向けエラーの詳細を、キーボードだけで開閉できる。
- [ ] 利用者向けエラーに、失敗内容、データ安全性、次の操作が表示される。
- [ ] プロジェクト、主対象文書、参考資料、テンプレート、Copilot用ZIPのダイアログに、承認済みタイトルとフィルター名が表示される。
- [ ] 表示倍率100%で、ボタン、入力項目名、エラー文が欠けない。
- [ ] 表示倍率150%で、ボタン、入力項目名、エラー文が欠けない。
- [ ] 表示倍率200%で、ボタン、入力項目名、エラー文が欠けない。
- [ ] 長い日本語エラー文を表示しても、内容をスクロールでき、操作ボタンが隠れない。
- [ ] `.clmproj`を保存して再度開いても、プロジェクト内容が変わらない。
- [ ] `.clmcheck`を保存して再度開いても、テンプレート内容が変わらない。
- [ ] Copilot用ZIPの内容とPythonバリデータが、文言変更前の契約を維持している。

## 証跡

手動確認の実施後、次を記録してください。

- 操作経路
- 確認した表示倍率
- 使用したテストデータ
- 発見した問題と再現手順
- 修正コミットまたは既知の制約

機密文書、個人情報を含むファイル名、絶対パス、文書本文は記録しないでください。
